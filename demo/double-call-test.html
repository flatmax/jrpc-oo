JRPC Double Call Test

About This Test

This test demonstrates bidirectional communication between a web client and Python JRPC server. The client can call server methods, and the server can call back to client methods, creating a "double call" pattern useful for real-time applications.

Instructions

  1. Make sure the Python server is running: python3 jrpc_oo/tests/JRPCServerTestDoubleCall.py
  2. The server runs on port 9001 by default
  3. If using HTTPS/WSS, ensure browser security clearance for the WebSocket connection
  4. Use the buttons below to test different functionalities:
    • Test Button Press: Simple server method call
    • Test Double Call: Server method that calls back to client
    • Get Server Status: Retrieve current server state
    • Reset Counter: Reset the server's call counter
